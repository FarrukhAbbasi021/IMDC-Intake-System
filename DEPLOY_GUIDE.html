Set up GitHub

~5 minutes
1
Create a free GitHub account
▼

GitHub is where your website files are stored. Think of it like Google Drive, but specifically for websites.

  1. Open your browser and go to https://github.com
  2. Click the big green "Sign up" button
  3. Enter your email address
  4. Create a password (write it down!)
  5. Choose a username — e.g. imdcforms
  6. Complete the puzzle verification
  7. Check your email and click the verification link
✓ You only need to do this once. After this, GitHub is free forever for public sites.
2
Create a new repository (your project's home)
▼

A "repository" is just a folder on GitHub that holds your website files.

  1. After logging into GitHub, click the + button in the top-right corner
  2. Select "New repository"
  3. In Repository name, type: imdc-intake-system
  4. Under visibility, select Public ← this is required for the free Render tier
  5. Do NOT check any of the "Initialize" boxes — leave them all unchecked
  6. Click the green "Create repository" button
ℹ️ You'll land on an empty repository page. That's correct — it's empty because we haven't uploaded anything yet.
3
Upload your project files to GitHub
▼

First, extract (unzip) the imdc-deploy.zip file on your computer. Then:

  1. On your empty repository page, look for the text "…or upload an existing file" — click "uploading an existing file"
  2. A drag-and-drop area appears
  3. Open the unzipped imdc-deploy folder on your computer
  4. Select ALL files and folders inside it: render.yaml, README.md, .gitignore, and the public folder
  5. Drag them all into the GitHub upload area
  6. Wait for them to finish uploading (you'll see a list appear)
  7. At the bottom of the page, in the "Commit changes" box, type: Initial upload
  8. Click the green "Commit changes" button
⚠️ Make sure you upload the contents of the folder, not the folder itself. GitHub should show files like render.yaml and a folder called public at the root level.

After upload, your repository should look like this:

github.com/your-username/imdc-intake-system
📁 public/
   📄 index.html
   📄 sw.js
   📄 _redirects
📄 .gitignore
📄 README.md
📄 render.yaml

Deploy on Render

~5 minutes
4
Create a free Render account
▼

Render is the hosting service that will take your files from GitHub and make them available as a real website.

  1. Go to https://render.com
  2. Click "Get Started for Free"
  3. Click "Continue with GitHub" — this is the easiest option. It links your GitHub account automatically.
  4. GitHub will ask if you allow Render access — click "Authorize render"
  5. You may need to verify your email address for Render
✓ Using "Continue with GitHub" saves you from having to enter another password. Render uses GitHub to know where your files are.
5
Create a new Static Site on Render
▼
  1. In the Render dashboard, click the "New +" button (top-right)
  2. From the dropdown, select "Static Site"
  3. Render will show you a list of your GitHub repositories
  4. Find imdc-intake-system and click "Connect"
ℹ️ If you don't see your repository listed, click "Configure account" and grant Render access to it.
6
Configure the deployment settings
▼

Render shows a form with settings. Fill them in exactly like this:

SettingWhat to enterWhy
Nameimdc-intake-systemThe name shown in Render dashboard
BranchmainWhich branch of your GitHub repo to use
Root Directoryleave blankUse the whole repository
Build Commandleave blankNo build needed — it's plain HTML
Publish DirectorypublicThis is the folder containing index.html ← most important
⚠️ The Publish Directory must be set to public (lowercase). If this is wrong, your site will show "Not Found".

After filling in the settings, scroll down and click the green "Create Static Site" button.

7
Wait for the deployment to complete
▼

Render now copies your files and makes them live. This takes about 1–2 minutes.

You'll see a log with lines scrolling by. Wait until you see:

dashboard.render.com
==> Deploying...
==> Uploading build...
==> Your site is live 🎉
   https://imdc-intake-system.onrender.com
🎉 When you see "Your site is live" — you're done! Click the URL shown to open your live form.

Test & Verify

~3 minutes
8
Test the live site — checklist
▼

Open your live URL and verify these things work:

  • ✓ The page loads and shows the IMDC header with navigation tabs
  • ✓ Fill in Practice Name on the Master Form — it should auto-fill on Account Setup tab
  • ✓ Click "Save & Populate Sub-Forms" — toast notification appears at bottom-right
  • ✓ Refresh the page — your data should still be there (localStorage working)
  • ✓ Open on your mobile phone — the form should be responsive
  • ✓ On mobile: tap "📷 Take Photo / Scan" in Documents tab — camera should open

  • ✓ Turn Wi-Fi back on — the bar should disappear and show "Synced"
  • ✓ URL starts with https:// — padlock icon in browser bar confirms secure

How to Update the Site Later

~2 minutes per update
10
Update via GitHub website (no coding needed)
▼

Every time you push a change to GitHub, Render automatically redeploys your site in about 60 seconds.

  1. Go to your repository on GitHub: github.com/your-username/imdc-intake-system
  2. Click on the public folder
  3. Click on index.html
  4. Click the pencil icon ✏ (top-right of the file view) to edit
  5. Make your changes
  6. Scroll down, write what you changed in the commit message box
  7. Click "Commit changes"
  8. Within 60 seconds, your live site updates automatically
✓ You can also replace the entire file: click the file → click the ⬆ upload icon → upload your new index.html → commit.
11
Add a custom domain (optional)
▼

Instead of imdc-intake-system.onrender.com, you can use forms.yourclinic.com.

  1. In Render dashboard → click your site → click Settings
  2. Scroll to Custom Domains → click Add Custom Domain
  3. Type your domain, e.g. intake.yourclinic.com
  4. Render shows you a CNAME record — copy it
  5. Log into your domain registrar (GoDaddy, Namecheap, Google Domains, etc.)
  6. Go to DNS Settings → Add Record → Type: CNAME
  7. Name: intake  |  Value: paste the value Render gave you
  8. Save and wait 10–60 minutes for it to propagate
  9. Render auto-provisions a free SSL certificate — the padlock appears automatically

Troubleshooting

?
Common problems and how to fix them
▼
ProblemFix
Site shows "Not Found" or blank page In Render → Settings → check Publish Directory is set to public (not Public or ./public)
My changes on GitHub aren't showing on the live site Hard refresh: press Ctrl+Shift+R (Windows) or Cmd+Shift+R (Mac). Also check Render dashboard — it should show a new deploy happened.
Render dashboard shows "Build failed" Click the failed deploy to see the error log. Usually means public/index.html is missing from the repository. Re-upload the files.
Repository doesn't appear in Render's list Click "Configure account" in Render → under Repository access, switch from "Only select repositories" to "All repositories", then save.
Camera not working Camera requires HTTPS. Only works on the Render URL (starts with https://). Does not work when opening index.html directly from your computer as a file.
Form data disappeared after refresh The browser's localStorage was cleared (e.g., cleared browser history or private/incognito mode). Use the Export function to save a backup JSON file regularly.
Can't drag files into GitHub upload area Try a different browser (Chrome works best). Or click "choose your files" to use a file picker instead of drag-and-drop.
Site is slow to load the first time Free Render sites "sleep" after 15 minutes of inactivity. The first load takes 30–60 seconds to "wake up". After that, it's fast. Upgrade to a paid Render plan to avoid this.
ℹ️ For more help: Render docs at render.com/docs — or GitHub docs at docs.github.com
